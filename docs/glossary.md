# Glossary

## A

**API (Application Programming Interface)**
A set of protocols and tools that allows different software applications to communicate with each other.

**Attestation**
A cryptographic proof that verifies the integrity and authenticity of a compute node's hardware and software configuration.

## B

**Batching**
Grouping multiple tasks together for more efficient processing.

**Blockchain**
A distributed digital ledger that records transactions across many computers in a way that ensures security and transparency.

## C

**Consensus Mechanism**
A protocol that ensures all nodes in a distributed network agree on the state of the system.

**Custom Node**
A specialized compute node that provides unique capabilities beyond standard AI inference.

## D

**DAO (Decentralized Autonomous Organization)**
An organization represented by rules encoded as a computer program that is transparent and controlled by organization members.

**Decentralized**
A system where control and decision-making are distributed across multiple nodes rather than centralized.

**Delegation**
The act of transferring staking rights to another node operator.

## E

**Epoch**
A fixed time period (6 hours in Synapse) during which rewards are calculated and distributed.

**Escrow**
A financial arrangement where a third party holds and regulates payment of funds required for two parties involved in a given transaction.

**Execution Environment**
The context in which code runs, including hardware resources and security constraints.

## F

**Federated Learning**
A machine learning approach where models are trained across decentralized devices using local data.

**Fraud Proof**
Evidence submitted to the network proving that a node executed a task incorrectly.

## G

**Gateway**
The entry point for client requests into the Synapse network.

**Governance**
The process by which token holders vote on protocol changes and decisions.

**GPU (Graphics Processing Unit)**
Specialized hardware optimized for parallel computation, ideal for AI/ML workloads.

## H

**Hardware Attestation**
Verification that a node has the claimed hardware capabilities.

**Heartbeat**
Regular status signals sent by nodes to prove they are online and operational.

## I

**Inference**
The process of using a trained AI model to make predictions on new data.

**IPFS (InterPlanetary File System)**
A distributed file system for storing and sharing data in a decentralized network.

## L

**Layer 2 (L2)**
A secondary framework built on top of an existing blockchain (Layer 1) to improve scalability.

**Latency**
The time delay between a task request and its completion.

## M

**Marketplace**
The platform where compute buyers and sellers interact.

**Model**
A trained AI system capable of performing specific tasks.

## N

**Node**
A computer participating in the Synapse network by providing compute resources.

**Node Operator**
The individual or organization running a compute node.

## P

**Proof-of-Compute (PoC)**
A mechanism where nodes prove they have completed computational work.

**Proof-of-Stake (PoS)**
A consensus mechanism where validators stake cryptocurrency to secure the network.

**Proof-of-Compute-Stake (PoCS)**
Synapse's hybrid consensus combining staking with verifiable computation.

## R

**Remote Attestation**
Verification of a node's TEE from a remote location.

**Reputation Score**
A metric representing a node's historical reliability and performance.

**Reward**
Payment earned by node operators for completing tasks.

## S

**Scheduler**
The component that assigns tasks to appropriate compute nodes.

**SDK (Software Development Kit)**
A collection of tools and libraries for building applications.

**Slashing**
The penalty mechanism that removes staked tokens from misbehaving nodes.

**Smart Contract**
Self-executing code on the blockchain that automatically enforces agreements.

**Stake**
Cryptocurrency locked as collateral to participate in network operations.

**SYN**
The native utility token of the Synapse network.

## T

**Task**
A unit of work submitted to the Synapse network for execution.

**TEE (Trusted Execution Environment)**
A secure area of a processor that ensures code and data are protected.

**Tokenomics**
The economic model and design of a cryptocurrency token.

## U

**Unstaking**
The process of withdrawing staked tokens from the network.

**Uptime**
The percentage of time a node is operational and available.

## V

**Validator**
A node responsible for verifying transactions and maintaining network consensus.

**Verification**
The process of confirming that a task was executed correctly.

**VRAM (Video RAM)**
Memory dedicated to graphics processing, important for GPU compute.

## W

**Wallet**
A software application for storing and managing cryptocurrency.

**WebSocket**
A protocol enabling real-time, bidirectional communication between clients and servers.

## Z

**zk-SNARK (Zero-Knowledge Succinct Non-Interactive Argument of Knowledge)**
A cryptographic proof allowing one party to prove they know a value without revealing the value itself.

**zk-STARK (Zero-Knowledge Scalable Transparent Argument of Knowledge)**
A type of zero-knowledge proof that is scalable and doesn't require a trusted setup.

**Zero-Knowledge Proof**
A cryptographic method allowing one party to prove to another that a statement is true without revealing any information beyond the validity of the statement.
