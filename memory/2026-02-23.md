# 2026-02-23 — Synapse MVP Complete

## What We Built

Completed full Synapse MVP — decentralized AI inference network with 6 major components:

### 1. Fixed /stats Contract (gateway-api)
- Changed field names: `nodes` → `nodes_online`, added `jobs_total`, `tokens_today`, `tokens_total`
- `updated_at` now returns ISO format string
- Added `getTokensToday()` to usage DB module
- 2-second cache preserved for performance

### 2. Hardened Installer (scripts/install.sh)
- GHCR image publishing: `ghcr.io/ayrtonmansi/synapse-node-agent:latest`
- OS detection (Linux/macOS/Windows)
- Network fallback: Linux uses `--network host`, macOS/Windows uses bridge + `host.docker.internal`
- 30s verification step that polls router/stats for node registration
- Added `scripts/install-smoke.sh` for installer verification

### 3. Router Reliability (services/router)
- Explicit routing weight formula: `success_rate * health / (1 + latency_penalty) / price_factor`
- Aggressive penalties: 0.5x health on failure, 0.3x on timeout/disconnect
- Tracks successRate, timeouts, lastError, avgLatencyMs (rolling 10-sample window)
- pendingJobs cleanup on node disconnect/timeout

### 4. Node Receipts (services/node-agent + gateway-api)
- Node generates `prompt_hash` and `output_hash` for each job
- Receipt JSON stored in SQLite: `{job_id, prompt_hash, output_hash, ts}`
- Database migration added for receipt fields
- Anti-fraud foundation for future ERC20 settlement

### 5. vLLM Model Profile (services/node-agent)
- `MODEL_PROFILE=vllm-deepseek-v3` forwards to local vLLM at `VLLM_URL`
- Graceful fallback to echo-stub if vLLM `/health` fails
- Node only advertises `deepseek-v3` when vLLM is available

### 6. Extended Smoke Test (scripts/smoke-test.sh)
- Verifies all 7 /stats fields are non-null
- Email + wallet API key generation tests
- 10 concurrent requests, >=9 must pass
- Validates jobs_today increments correctly
- Confirms nodes_online >= 1
